Shrinking the Cross Section.pdf

# Summary: "Shrinking the Cross Section" (Kozak, Nagel, Santosh 2017)

## 1. Main Research Question
Can a large set of stock characteristics jointly explain cross-sectional return variation better than sparse factor models (e.g., Fama-French 5-factor)? And how do you estimate this without overfitting?

## 2. ML Methods Used
**Bayesian shrinkage / ridge regression variant** applied to SDF coefficient estimation. Standard OLS overfits badly with hundreds of characteristics, so they impose an economically motivated L2 penalty — shrinking coefficients of low-variance principal components more aggressively than high-variance ones. This is ridge regression but with the penalty tied to Sharpe Ratio plausibility rather than raw coefficient size.

## 3. Most Predictive Features
No single characteristic dominates. The key finding is that **principal components of the full characteristics universe** matter collectively. High-eigenvalue PCs (capturing broad market variance) are most important; low-eigenvalue PCs get heavily shrunk. Sparse models using only a handful of characteristics (B/M, profitability, investment) are demonstrably insufficient.

## 4. Key Performance Findings
- Characteristics-sparse SDFs (4–6 factor models) **fail** when confronted with the full predictor universe
- A **relatively small number of PCs** of the full factor set approximates the SDF well out-of-sample
- Heterogeneous shrinkage (penalizing low-variance PCs more) materially improves OOS performance vs. uniform shrinkage

## 5. What to Steal for Our LightGBM Model

| Insight | Application |
|---|---|
| Don't limit features to a sparse factor model | Use broad characteristics universe; let regularization do the pruning |
| Penalize low-variance features more heavily | Weight feature importance by variance contribution during selection |
| Use PCA on correlated factors | Pre-process correlated characteristics via PCA before feeding LightGBM |
| Ridge-style regularization outperforms OLS | Tune `lambda_l2` aggressively in LightGBM hyperparameter search |

**Bottom line:** Cast a wide feature net, apply strong regularization, and trust PCA to surface the signal.

Dissecting Characteristics Nonparametrically.pdf

# Summary: Dissecting Characteristics Nonparametrically (Freyberger, Neuhierl, Weber 2017)

## 1. Research Question
Which stock characteristics provide **independent, incremental information** for predicting cross-sectional expected returns, without imposing linear functional-form assumptions?

## 2. ML Methods Used
**Adaptive Group LASSO** with nonparametric estimation via quadratic splines. Chosen because it:
- Handles 36+ characteristics simultaneously without curse of dimensionality
- Performs automatic feature selection (shrinks irrelevant predictors to zero)
- Allows flexible, nonlinear functional forms
- Is robust to outliers (unlike OLS-based Fama-MacBeth regressions)

## 3. Most Predictive Characteristics
From 36 candidates, only **15 survive** selection for full sample:
- **Size** (market cap)
- **Idiosyncratic volatility**
- **Past return-based predictors** (momentum family)
- **Standardized unexplained volume** (for large-cap stocks)
- 7 characteristics remain significant for large-cap stocks (above NYSE 20th size percentile)

## 4. Model Performance
- In-sample Sharpe ratio: **~3.0** (all stocks), **1.81** (large caps)
- Out-of-sample Sharpe ratio: **3.42** vs. 2.26 for linear model — a **~50% improvement**
- Linear model selected 21 features but performed worse OOS, confirming overfitting risk from irrelevant characteristics

## 5. What to Steal for Our LightGBM Model

| Insight | Application |
|---|---|
| **Nonlinearities matter** | LightGBM naturally captures these — don't force linear transforms |
| **Aggressive feature selection** | Start with size, idiosyncratic vol, momentum; prune weak features |
| **OOS rolling-window validation** | Use expanding/rolling windows, predict one month ahead |
| **Hedge portfolio Sharpe as eval metric** | Supplement RMSE with long-short portfolio performance |
| **Beware characteristic zoo** | More features ≠ better; regularize aggressively via `min_child_samples`, `lambda_l1` |

# Summary: Empirical Asset Pricing via Machine Learning (Gu, Kelly, Xiu 2020)

## 1. Main Research Question
Can machine learning methods better measure equity risk premiums (expected returns) than traditional regression-based approaches, both cross-sectionally and in time series?

## 2. ML Methods Used
The paper compares multiple methods including **tree-based models** (most relevant to us), penalized regressions (LASSO, Ridge, Elastic Net), and dimension reduction approaches. Trees were favored because they naturally capture **nonlinear interactions** between predictors — something linear models systematically miss. Regularization was applied across all methods to prevent overfitting on high-dimensional feature sets.

## 3. Most Predictive Features
All methods converged on the same dominant signals:
- **Momentum** (price trend variables)
- **Liquidity** (trading volume, bid-ask related)
- **Volatility** (return variance measures)

These three feature families consistently dominated across model types — a strong consensus signal.

## 4. Key Performance Findings
- ML forecasts delivered large **out-of-sample economic gains**
- Long-short decile spread strategies achieved **Sharpe ratios exceeding 1.35** — more than double leading regression benchmarks
- Predictive gains traced directly to **nonlinear predictor interactions** that linear models miss

## 5. What to Steal for Our LightGBM Model

| Takeaway | Implementation |
|----------|---------------|
| Prioritize momentum, liquidity, volatility features | Build multiple variants of each signal |
| Nonlinear interactions are the alpha source | LightGBM captures these natively — trust the splits |
| Regularization is critical | Tune `min_child_samples`, `lambda_l1`, `num_leaves` aggressively |
| Use out-of-sample R² as primary validation metric | Avoid in-sample overfitting traps |
| Long-short decile spread Sharpe as economic metric | Evaluate portfolios, not just model accuracy |

**Bottom line:** This paper validates tree-based models as top performers. Focus feature engineering on momentum, liquidity, and volatility variants — that's where the edge lives.

Factor Momentum Everywhere JPM Quant 19.pdf

# Factor Momentum Everywhere — Summary for LightGBM Team

## 1. Main Research Question
Do common equity factors (value, momentum, quality, etc.) themselves exhibit momentum? Can timing factor exposure based on recent factor performance generate alpha beyond traditional stock momentum?

## 2. ML Methods Used
This is a **quantitative finance paper, not ML-focused**. Methods are traditional factor analysis, time-series regressions, spanning regressions, and Sharpe ratio comparisons across 65 characteristic-based factors. No ML techniques to report.

## 3. Most Predictive Features/Characteristics
- **Prior 1-month factor return** is the single strongest predictor of near-term factor performance (AR(1) ≈ 0.11 average across factors)
- **12-month and 5-year look-back windows** also add value, showing factor momentum is durable across horizons
- 65 characteristic-based factors including value, momentum, quality, and profitability variants

## 4. Key Performance Findings
- Time-Series Factor Momentum (TSFM) achieves **Sharpe ratio of 0.84**, beating stock momentum UMD (0.56)
- Performance holds **net of transaction costs**
- Results are **global** — replicated in international equity markets
- Factor momentum is **complementary to stock momentum**, not a substitute

## 5. What to Steal for Our LightGBM Model

| Feature Idea | Implementation |
|---|---|
| **Short-term factor return** | 1-month rolling return for each factor exposure (value, momentum, quality) as a feature |
| **Multi-horizon factor momentum** | Add 1M, 12M, 60M factor return features — signals persist across horizons |
| **Factor momentum × value interaction** | Combining factor momentum + stock momentum + value improves portfolio outcomes |
| **Cross-sectional factor rank** | Rank each stock's factor exposures relative to peers as additional features |

**Bottom line:** Add lagged 1-month returns of factor portfolios (HML, RMW, CMA, UMD) as features in LightGBM — these carry incremental predictive signal beyond individual stock characteristics alone.

# Summary: Rossi (2018) — Stock Return Prediction with Boosted Regression Trees

## 1. Main Research Question
Can a semi-parametric ML method (BRT) outperform standard linear models in forecasting monthly stock returns and volatility using large information sets?

## 2. ML Methods Used
**Boosted Regression Trees (BRT)** — chosen because it:
- Handles large feature sets without the curse of dimensionality
- Captures **non-linear and non-monotonic** relationships that linear models miss
- Avoids overfitting via shrinkage, subsampling, and model averaging (boosting ensemble logic)
- Imposes no parametric assumptions on functional form

This is directly analogous to **LightGBM**, which uses the same gradient boosting on decision trees framework.

## 3. Most Predictive Features
The paper uses the standard Goyal-Welch predictor set, including:
- **Valuation ratios** (dividend yield, earnings yield, book-to-market)
- **Interest rate variables** (short/long-term treasury spreads, corporate bond spreads)
- **Corporate financing activity** (payout ratios, aggregate issuance)
- **Macroeconomic variables** (inflation, industrial production)

Key insight: these predictors matter most through **non-linear interactions**, not linear effects.

## 4. Model Performance
- BRT **outperforms linear benchmarks** on both MSE and directional accuracy out-of-sample
- Generates **profitable portfolio allocations** even after transaction costs
- Confirms that predictor-to-return relationships are highly non-linear and non-monotonic

## 5. What to Steal for Our LightGBM Model
| Takeaway | Implementation |
|---|---|
| Use the full Goyal-Welch macro/valuation predictor set | Add dividend yield, term spread, credit spread as features |
| Don't force linearity — let trees find interactions | ✅ LightGBM handles this natively |
| Use **expanding window** out-of-sample validation | Avoids data snooping |
| Evaluate on **directional accuracy**, not just MSE | Add classification metric alongside regression |
| Apply **shrinkage + subsampling** | Tune `learning_rate`, `subsample`, `min_child_samples` |
